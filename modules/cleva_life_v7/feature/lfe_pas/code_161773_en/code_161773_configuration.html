    • Cleva Outputs

      This module supports integration between Cleva software and third-party applications regarding output management, allowing document generation, sending and archiving.

    • Cleva Party management

      This module manages the party management functional area, common to Life and Non-life. In particular, this module is a software layer that integrates with Cleva Common area v6, Cleva Common area v7 and Cleva Life v7.

    • Cleva Process factory

      This module allows insurers to define their own business processes. Based on the concept of product library, which has several components integrated with the Cleva software, the insurer is provided with accelerators to address business processes such as underwriting and claims.

    • Cleva Product factory

      This module includes product engineering features, which allow the insurance company to configure all the components the system needs to manage a product, including tariffs, data validation rules, risk acceptance, calculation of taxes and commissions, claim context, indemnity calculation rules and the behaviour expected in a specific distribution channel.

    • Cleva Security manager

      This framework supports features associated with security and roles. It allows insurers to create profiles and to define which features shall apply to them.

    • Cleva Technical accounting

      This module supports the integration of the insurance company’s technical data with the corresponding accounting system. This means that, with this module, it is possible to process the documents/events produced by the different operations and automatically integrate them in the accounting system.

    • Cleva Workflow

      This module includes the workflow engine that orchestrates all business processes and corresponding insurance activities, from underwriting to claim management. It allows implementing activity automation according to the business rules defined by the insurance company, and the activities processed manually are available to users via the task manager.

Cálculo de encargos de recibo por esquema de cálculo

Esta funcionalidade tem como objetivo permitir que o cálculo de encargos de recibo seja efetuada através de esquema de cálculo e permitir a configuração de encargos por cobertura.

Para configurar o calculo de encargos por esquema, será necessário efetuar as configurações com a informação abaixo descrita.

CODE-161773 | Cleva Life v7 | 7.11 | PAS | All markets

  2. Associar esquema de calculo de encargos ao produto

    Para efetuar esta configuração, comece por aceder a General class _Manage calculation by class.

    Na área apresentada indique o produto e o evento de calculo de encargos ('EN').

    De seguida, insira ou altere o esquema de calculo pretendido.

  3. Definição de encargos por tipificação de cobertura e/ou cobertura

    Existindo necessidade de definição de encargos por tipificação de cobertura e/ou cobertura, será necesário configurar esses valores considerando a seguinte informação:

    • Tipo de encargo
      • 'P' - Percentagem
      • 'V' - Valor
    • Nº sequencia do encargo
      • 1 a 5
    • Tipificação da cobertura
      • '01' – AGRAVAMENTO
      • '02' – MORTE
      • '03' – VIDA
      • '04' – INVALIDEZ
      • '07' - PARTICIPACAO RESULTADOS
      • '08' - Não Vida / Saúde
      • '09' - Não Vida / Assistência ao Lar
      • '10' - Não Vida / Outras assistências
    • Cobertura
    • Data efeito
    • Percentagem
      • Apenas para encargos por percentagem
    • Valor
      • Apenas para encargos por valor
    Notas:
    • A configuração terá que ser efetuada por tipificação de cobertura ou por cobertura.
      • Se configuração por tipificação de cobertura, a indicação da tipificação da cobertura é obrigatoria e a cobertura deverá ter o valor -1.
      • Se configuração por cobertura, a indicação da cobertura é obrigatoria e a tipificação da cobertura deverá ter o valor ''.
    • Nesta versão não existe ainda processo interativo para configurar esta informação, tendo a mesma que ser efetuada diretamente na base de dados (Tabela: MODENCCBT).
