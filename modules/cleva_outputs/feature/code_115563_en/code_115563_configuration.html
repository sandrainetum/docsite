  • Module guides

    This section provides all module guides organized by module.

    • Cleva Analytics

      This module includes several data marts generated based on the business of Life, Non-life and Accounting areas. It allows extracting, from massive data volumes, information that is relevant for a quick decision-making process.

    • Cleva Frontend factory

      This module provides features to define the UX/UI of all Cleva software. It allows insurers to define menus, pages and sections, and to integrate different Cleva modules within the same user interface.

    • Cleva Outputs

      This module supports integration between Cleva software and third-party applications regarding output management, allowing document generation, sending and archiving.

      • General information

        This chapter introduces overview information about Cleva Outputs, such as its architecture, roles and other resources.

      • Flow
      • Enrichers
      • Document generation

        This chapter refers to the phase when the document is generated, based on data merged layout. This step may also be a batch operation, in which case no return is expected.

      • Document archiving

        This chapter refers to the document management, where it is possbile, for example, to archive a document or search for stored documents.

      • Configurations
        • Flow configurator

          This feature provides a console that allows configuring the flow steps and actions for each event that might generate an output.

        • Data purge

          This feature provides a routine process that allows purging the output's data according to a time condition.

    • Cleva Product factory

      This module includes product engineering features, which allow the insurance company to configure all the components the system needs to manage a product, including tariffs, data validation rules, risk acceptance, calculation of taxes and commissions, claim context, indemnity calculation rules and the behaviour expected in a specific distribution channel.

    • Cleva Technical accounting

      Cleva Technical accounting (TAC) is a solution designed to support the integration of the insurance company’s core system with its accounting system.

    • Cleva Workflow

      This module includes the workflow engine that orchestrates all business processes and corresponding insurance activities, from underwriting to claim management. It allows implementing activity automation according to the business rules defined by the insurance company, and the activities processed manually are available to users via the task manager.

Flow configurator

This feature provides a console that allows configuring the flow steps and actions for each event that might generate an output.

The activation of this feature requires the configurations below.

CODE-115563 | Cleva Outputs | 7.10 | All markets

  • Process configuration

    To configure a process, follow the steps below.

    1. Create new process

      Go to the process configuration area in Configuration _ Processes.

      To create a new process, click Create process and complete the fields displayed, as indicated below.

      • Name: Name of the process.
      • Description: Description of the process.
      • Context: Context of the process with the following options:
        • Archive: The process allows archiving;
        • Outputs: The process allows outputs;
        • Communication: The process allows communications.

      Finally, click Create.

      You can also search a process already configured, using its name and context.

    2. Create the documents associated with the process

      After creating the process, click Manage in the process. You can edit the process, remove it and create documents associated with it.

      To create a document, click Create document and complete the fields displayed, as indicated below.

      • Id of document type: Id of the document type.
      • Process: Name of the process linked to the event.
      • Confidentiality: Specifies whether the document is confidential or not.
      • Outputs: Specifies whether the document allows outputs or not. If it does not, then the document will not be available if you want to add it to the event linked to this process, as you will see later in this document.
      • Description: Description of the document type.
      • Comments: Comments.

      Then, click Next and select the supported formats for the type of document. The following formats are available:

      • All – all formats are supported;
      • .pdf;
      • .jpeg;
      • .jpg;
      • .bmp;
      • .png;
      • .bpm;
      • .doc;
      • .xls;
      • .xlsx;
      • other – you can choose another format.

      Then, click Next and select the document type metadata. Metadata is data that provides information about other data. It is used for discovery and identification. For example, if you want to know more information about a receipt, you must specify that the RECEIPT_ID metadata is mandatory. However, the more metadata you provide, the more chances you have of finding the specific receipt.

      Click Save. The document is created.

      You can create as many documents as necessary.

  • Attributes configuration

    To configure an attribute of a certain type, follow the steps below.

    1. Create attribute
      Go to the attribute configuration area, in Configuration _ Attributes

      To create an attribute, click the Create attribute button and complete the fields displayed, as indicated below.

      • Type of attributes: Type of the attribute. You can choose if the attribute will be of type enricher, dispatch or notification.
      • Activate attribute: Attribute is active if selected.

      If you select the notification type attribute, complete the fields displayed as indicated below.

      • Type: Type of the attribute with the following options:
        • QUEUE;
        • TOPIC.
      • Description: Attribute description.
      • Name: Attribute name.
      • Target: Attribute target.

      If you select the dispatch type attribute, complete the fields displayed as indicated below.

      • Type: Name of the dispatch type attribute.
      • Description: Attribute description.

      If you select the enricher type attribute, complete the fields displayed as indicated below.

      • Type: You can choose EJB or REST type.
      • Description: Attribute description.
      • Name: Attribute name.
      • Target: Attribute target.
      • User name: User name.
      • Password: Password.

      Then click Save and the attribute is created.

      You can search an attribute already configured, using its type and name.

  • Events configuration
    To configure an event, follow the steps below.
    1. Create a new event linked to an existing process

      To create an event, click Add event and complete the fields displayed, as indicated below.

      • Event: Event name. The event name must be different from the existing event names.
      • Process: Select the process you want to link to the event. Only existing processes should appear in the process list.
      • Expiration time: Specifies how long documents associated with this event will be available after being generated. '0 days' indicates that generated documents are never deleted.
      • Description: Description of the event.
      • Active actions:
        • Enricher: The enricher concept exists so that the output data can be enriched with more information before the output is generated. This can be used to inject in the request some specific information that needs to be sent to the output generation.
        • Get format: Prints the layout that is configured for the document associated with the event. Note that in the document this Get format action must also be active so that it is possible to print the desired layout, otherwise a PDF will be printed, with only the information coming from the request.
        • Archive: It is possible to archive documents in the document management area.
        • Notify: It sends the notification that is associated with the event. Note that in the document this Notify action must also be active so that it is possible to send the notification.
        • Save: It is possible to transfer the generated document in the output console (Output menu).
        • Immediate execution of the batch process: It is possible to transfer the generated document in the outputs console (Output menu) in cases where a batch request is made. For this to work the Save action must also be active.

      Then click Save and the event is created.

      You can remove the event or manage it.

      After creating the event, click Manage in the event.

      You can edit the event, add documents to it in the Documents tab, create a new enricher or associate an existing enricher with the event in the Enrichers tab and create a new notification or associate an existing notification with the event in the Notifications tab.

    2. Create a document associated with an event

      In the Documents tab, click Add and complete the fields displayed, as indicated below.

      • Id of document type: Here, only documents created in the process linked to this event with an active outputs option are available for selection.
      • Method of dispatch:
        • Only data: The shipped content comes in xml format.
        • Complete content: The shipped content comes in PDF format.
      • Active actions:
        • Get format: Prints the layout that is configured for the document associated with the event. Note that in the event this Get format action must also be active so that it is possible to print the desired layout, otherwise a PDF will be printed, with only the information coming from the request.
        • Index: It indexes the document.
        • Archive: It is possible to archive documents in the document management area.
        • Dispatch: It sends the dispatch that is associated with the document associated with the event.
        • Notify: It sends the notification that is associated with the event. Note that in the document this Notify action must also be active so that it is possible to send the notification.
        • Transform: Gets the content that will be sent to the outputs in xml if not selected or converts it to JSON if selected.

      Then, click Save and the document associated with the event is created.

      After creating the document, you can manage the document or delete it.

    3. Add a dispatch to the document associated with an event

      After creating the document, click Manage in the document. To edit the document, click Edit.

      In the edit mode, in the Dispatches tab, click Add and complete the field displayed, as indicated below.

      • Description: Description of the dispatch.

      To add a new dispatch to the document, click Create and complete the fields displayed, as indicated below.

      • Type: Type of the dispatch.
      • Description: Description of the dispatch.

      Then, click Save. The new dispatch is created but it is not immediately associated with the document, since it is necessary to search for the newly created dispatch by completing the description, as shown below.

      Click Save. The dispatches are then associated with the document. Dispatches can then be removed from the document, if desired.

    4. Create a layout associated with a document associated with an event

      After creating the document, click Manage in the document. You can edit the document by clicking Edit.

      In edit mode, in the Layouts tab, click Add and complete the field displayed, as indicated below.

      • Name: Path where the layout is located
      • Layout type:
        • Generic: If you choose this option, then the indicated layout in the Name field is what is printed by default.
        • Product: If you choose this option, then you will have to specify the product (note: the product consists of four digits), and it will mean that, for that product, the layout that you specify in the Name field will be the one that will be printed.
        • Product class: If you choose this option, then you will have to specify the class of an existing product (note: the class consists of two digits), and it will mean that, for the products of this class, the layout that you specify in the Name field will be the one that will be printed.

      Then, click Save. The layout is now associated with the document.

      Click Save to save the changes made to the document.

      You can associate more than one layout with the document, which allows having a different layout for each product, for example. Layouts can then be removed from the document, if desired.

    5. Associate an enricher with an event

      In the Enrichers tab, click Associate to associate an enricher with an event.

      Then complete the field displayed, as indicated below.

      • Select an enricher: Here you can select an enricher already configured.

      Then, click Associate. The enricher is now associated with the event.

      In the Enrichers tab, click Create to create and at the same time associate an enricher with the event. Then complete the fields displayed, as indicated below.

      • Name: Name of the enricher.
      • Description: Description of the enricher.
      • Type:
        • EJB;
        • REST;
      • Target: Target of the enricher. This field cannot contain spaces between letters.
      • User name: The user name.
      • Password: The password.

      Then click Create. The enricher is now created and associated with the event.

      To view details of the associated enricher, click Actions in the line corresponding to the enricher and then click View details.

      The enricher details are then displayed.

      To edit the associated enricher, click Actions in the line corresponding to the enricher and then click Edit enricher.

      The enricher edit window is displayed. Here you can make changes to any of the fields.

      To deactivate the associated enricher, click Actions in the line corresponding to the enricher and then click Deactivate.

      The Deactivate event enricher window is displayed. To deactivate the enricher, click Deactivate.

      The enricher is now deactivated.

      To remove the association between the enricher and the event, click Actions in the line corresponding to the enricher and then click Remove association.

      The Remove event enricher window is displayed. To remove the enricher, click Remove.

      The enricher is now removed.

    6. Associate a notification with an event

      In the Notifications tab, click Associate to associate a notification with an event.

      Then complete the field displayed, as indicated below.

      • Select a notification destination: Here you can select a notification already configured.

      Then click Associate. The notification is now associated with the event.

      In the Notifications tab, click Create to create and at the same time associate a notification with the event. Then complete the fields displayed, as indicated below.

      • Name: Name of the notification.
      • Description: Description of the notification.
      • Type:
        • QUEUE;
        • TOPIC;
      • Target: Target of the notification. This field cannot contain spaces between letters.

      Then, click Create. The notification is now created and associated with the event.

      To remove the associated notification from the event, click Remove from event in the line corresponding to the notification.

      The Remove event notification window is displayed. To remove the notification, click Remove.

      The notification is now removed.

  • Communications configuration

    To configure a communication, follow the steps below.

    1. Create new communication

      To create a communication, click Create communication and complete the fields displayed, as indicated below.

      • Name: Name of the communication.
      • Process: The list of processes that have the Communication option selected in the context field will appear.
      • Event: The list of events linked to the selected process will appear.
      • Document type: The list of documents associated with the process selected will appear.
      • Communication options:
        • Archive: The communication allows archive;
        • Dispatch: The communication allows dispatch;
        • Confidential: Specifies whether the communication is confidential or not.

      Then, click Next.

      Here you can drag and drop different form components to build the communication.

      Then, click Save and the communication will be created.

      Later you can edit or delete the communication.

      You can also search a communication already configured, using the Process and Communication attributes.
