Data purge

This feature provides a routine process that allows purging the output's data according to a time condition.

This configuration aims at creating a periodic process to purge information from Cleva Outputs.

To do this configuration, go to the Cleva Integration hub’s console.

Then go to Integration hub services _ Batch configuration and add the following batch configurations:

  • pt.i2s.cms.purge.batch.start.cronExpression – cron expression defining the job start;
  • pt.i2s.cms.purge.batch.endTime – the job stop time in format HHMMSS;
  • pt.i2s.cms.purge.batch.username – the default user name to use.

CODE-115573 | Cleva Outputs | 7.10 | All markets

  1. Configure the pt.i2s.cms.purge.batch.start.cronExpression batch
    1. To configure this batch, click +Add in the Batch configuration area.
    2. An Edit batch configuration pop-up is displayed so that you can add new batch configurations to the environment.
    3. In the Criteria stepper, complete the fields displayed, as indicated below.
      • Configuration name – pt.i2s.cms.purge.batch.start.cronExpression;
      • Description – cron expression defining the job start.
    4. To define the sequence, click Next.
    5. Indicate the frequency.

      In the example below, the frequency is set to daily, starting at 08:30:00.

    6. Click Save.

      The configuration is now saved successfully.

  2. Configure the pt.i2s.cms.purge.batch.endTime batch
    1. To configure this batch, click +Add in the Batch configuration area.
    2. An Edit batch configuration pop-up is displayed so that you can add new batch configurations to the environment.
    3. In the Criteria stepper, complete the fields displayed, as indicated below.
      • Configuration name – pt.i2s.cms.purge.batch.endTime;
      • Description – the job stop time in format HHMMSS;
      • Value – enter a value in format HHMMSS, according to the time desired for the batch to stop executing (example: 22h 30 min and 0 seconds).
    4. Click Save.
  3. Configure the pt.i2s.cms.purge.batch.username batch
    1. To configure this batch, click +Add in the Batch configuration area.
    2. An Edit batch configuration pop-up is displayed so that you can add new batch configurations to the environment.
    3. In the Criteria stepper, complete the fields displayed, as indicated below.
      • Configuration name – pt.i2s.cms.purge.batch.username;
      • Description – the default user name to use;
      • Value – enter the user name (example: XXX).
    4. Click Save.
    5. Then, configure the variables below, in Integration hub services _ Variable configuration.
      • pt.i2s.cms.external.output.api.batch.requests.purge.files.older.than.days – number of days to purge the CMS Outputs batch documents.
      • pt.i2s.cms.external.output.api.immed.requests.purge.files.older.than.days – number of days to purge the CMS Outputs immediate documents.
  4. Configure the pt.i2s.cms.external.output.api.batch.requests.purge.files. older.than.days variable
    1. To configure this variable, click +Add in the Variable configuration area.
    2. The Create variable pop-up is displayed so that you can add the new variables.
    3. In the Create variable pop-up, complete the fields displayed, as indicated below.
      • Application – CMS.
      • Variable – pt.i2s.cms.external.output.api.batch.requests.purge.files.older.than.days.
      • Value – 0.
      • Description – number of days to purge the CMS Outputs batch documents.
      • Instance – ALL.
        Note: The Position field is completed automatically.
    4. Click Save.
    5. Confirm the operation.
  5. Configure the pt.i2s.cms.external.output.api.immed.requests.purge. files.older.than.days variable
    1. To configure this variable, click +Add in the Variable configuration area.
    2. In the Create variable pop-up, complete the fields displayed, as indicated below.
      • Application – CMS.
      • Variable – pt.i2s.cms.external.output.api.immed.requests.purge.files.older.than.days.
      • Value – 0.
      • Description – number of days to purge the CMS Outputs immediate documents.
      • Instance – ALL.
    3. Click Save.
    4. Confirm the operation.
