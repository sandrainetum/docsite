  • Release documentation

    This section provides all documents produced for each module and version.

    • Cleva (all modules)

      Provides topics transversal to all Cleva modules.

    • Cleva Analytics

      This module includes several data marts generated based on the business of Life, Non-life and Accounting areas. It allows extracting, from massive data volumes, information that is relevant for a quick decision-making process.

    • Cleva Channels life v7

      This module supports the insurance portfolio management in the various channels and distribution devices of a life insurance company, such as banks or brokers, thus responding to the omnichannel concept. Its features cover the complete life cycle of the policy, from quotation to proposal and contract creation including risk management and policy acceptance associated with the company’s rules.

    • Cleva Common area v7

      This module includes features used by the different modules, such as batch console and notifications management. Additionally, this module also includes the functional areas of the module Cleva Common area v6 resulting from the Non-life modernization project.

    • Cleva Frontend factory

      This module provides features to define the UX/UI of all Cleva software. It allows insurers to define menus, pages and sections, and to integrate different Cleva modules within the same user interface.

    • Cleva Integration hub

      The main objective of this module is to allow the integration with the several systems of the insurance company. Therefore, Cleva Integration hub provides a set of features to manage web service integration.

    • Cleva Life v7

      This module accommodates all core features of a life insurance company, for any product or risk. Its features cover all insurance products, with particular reference to policy and endorsement management, commission management, claims and indemnity management, risk sharing management, coinsurance and reinsurance.

    • Cleva Outputs

      This module supports integration between Cleva software and third-party applications regarding output management, allowing document generation, sending and archiving.

    • Cleva Party management

      This module manages the party management functional area, common to Life and Non-life. In particular, this module is a software layer that integrates with Cleva Common area v6, Cleva Common area v7 and Cleva Life v7.

    • Cleva Process factory

      This module allows insurers to define their own business processes. Based on the concept of product library, which has several components integrated with the Cleva software, the insurer is provided with accelerators to address business processes such as underwriting and claims.

    • Cleva Product factory

      This module includes product engineering features, which allow the insurance company to configure all the components the system needs to manage a product, including tariffs, data validation rules, risk acceptance, calculation of taxes and commissions, claim context, indemnity calculation rules and the behaviour expected in a specific distribution channel.

    • Cleva Security manager

      This framework supports features associated with security and roles. It allows insurers to create profiles and to define which features shall apply to them.

    • Cleva Technical accounting

      This module supports the integration of the insurance company’s technical data with the corresponding accounting system. This means that, with this module, it is possible to process the documents/events produced by the different operations and automatically integrate them in the accounting system.

    • Cleva Workflow

      This module includes the workflow engine that orchestrates all business processes and corresponding insurance activities, from underwriting to claim management. It allows implementing activity automation according to the business rules defined by the insurance company, and the activities processed manually are available to users via the task manager.

      • 7.11
        • Feature list
          • Componentisation of listeners

            The structure of components in the Product libray was extended to support the characteristics of listeners.

          • Redesign of the widget configuration screen

            The widgets page in Framework configurator, to configure the widgets that are displayed for tasks and processes in the Task manager, was modified for a better user experience.

          • Communication between widgets in the Task manager

            This feature allows widgets to adapt their behaviour depending on shared information that is available for all widgets.

          • Cleva Widgets catalogue's translations

            This feature allows the translation of information regarding Cleva widgets.

          • Data cleanup

            This feature allows automatically deleting old information about processes.

          • New process category: Surrenders

            A new item was added to the list of available categories to allow categorising and filtering surrender processes. The name of this new category is Surrenders.

          • Multichannel support in the BAM component and in operational dashboards

            The Business Activity Monitoring (BAM) component and operational dashboards for tasks and SLA managements were updated to support the filter of information by channel.

          • Sort results by deadline when searching for processes

            The process search results are now sorted by deadline, by default.

          • Standardisation of table styles in Framework configurator

            To align styles in all screens and to promote a better user experience, the action buttons in the tables that are displayed in Framework configurator were changed into icons..

          • Comments on tasks

            A new widget is available to register and view comments in a task.

          • Distinguish process by identification document

            This feature modified the searches of Processes and Tasks in the Task manager to seek elements by the identification document value of the party instead of the tax id.

          • Configuration of process access out of the flow definition

            This feature allows configuring users and groups of users that have permissions to instanciate or access processes in a configuration screen.

          • Automatic widget configurations

            This feature allows creating configurations of widgets for processes using either the Cleva base configurations or a new mechanism for importing and exporting configurations.

          • Ad hoc configuration of widgets in the Task manager

            This feature allows controlling the behaviour of a widget in the Task manager.

          • Registration of Cleva Workflow jobs in the new batch console

            A new batch console was implemented for all Cleva modules in release 7.11.

          • Cleva process templates and components in Modeller

            The Modeller has been extended to support Cleva process templates and components.
