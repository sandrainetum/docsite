Componentisation of listeners

The structure of components in the Product libray was extended to support the characteristics of listeners.

One of the objetives of Cleva Workflow is to provide you with a Product library with enough elements that allow you to create your own processes. This approach encourages the reuse of these Product library elements, so that less code is required,and processes can be implemented in a timely fashion.

The Product library is currently composed of three elements: processes, components and widgets. This feature focuses on components. Components are backend elements that can be reused in process models.

Before this release, components were created to be used used in service tasks. When the flow went through one of these service tasks, the workflow engine automatically executed the associated component.

This feature introduces another way of executing backend elements, that allows using them as listeners. Listeners can be configured in different process model elements, such as process activities, plan items (CMMN elements) and tasks. When configuring a listener, an event or a status transition is defined and when that defined event or transition happens the listener's component is executed.

To allow the reuse of these listeners components, the structure of components was extended to support the characteristics of listeners. Therefore, a new type of component is available to defined listeners.

Additionally, the existing listeners that are agnostic to business functionalities were adapted to the listener component structure and included in the components catalogue.
