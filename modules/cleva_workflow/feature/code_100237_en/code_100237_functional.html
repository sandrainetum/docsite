SLA in processes and cases

This feature provides several improvements regarding the SLA.

This release introduces the capability of defining Service Level Agreements (SLA) for processes both modelled in BPMN and in CMMN. This feature completes the SLA functionality already available for tasks.

As part of the functionality, the screen for configuring SLA in the SLA menu in the framework configurator was redesigned to improve the user experience. Now, the SLA screen has two sections.

  1. Left panel for searching processes and tasks within processes. There are several filters available, so it is easier to find the process or task that the user is looking for.
  2. Central panel with either the process or the task information. After selecting either the process or the task, the information for the selected item is displayed. The information is split into the following sections.
    1. General information with the name of the selected process/task.
    2. SLA duration. This section displays the defined SLA duration. If it is not defined, it is possible to define it, and if it is already defined, it is possible to edit or delete it.
    3. Actions section. In this section, it is possible to configure actions to be executed when the duration of the process/tasks exceeds a defined time.
    4. Exclusion section. This section is only applicable to processes and displays the tree of subprocesses and tasks within that process. It is possible to select any of the items in the tree, so that the duration of the selected item is excluded from the SLA duration. This way, the SLA duration is not impacted by the duration of the excluded items.

In the task manager, the following modifications were done.

  1. SLA levels. These are levels that categorise any process and task as to the SLA. There are four SLA levels:
    1. Within the limits – Green colour. Progress of the SLA between 0% - 50%.
    2. At risk – Yellow colour. Progress of the SLA between 50% - 75%.
    3. Reaching deadline – Orange colour. Progress of the SLA between 75% - 100%.
    4. Overdue – Red colour. Progress of the SLA greater than 100%.
  2. Search processes. This screen was modified to display the SLA level, the due date, time to the due date and the elapsed percentage for the results.
  3. Search tasks. This screen was modified to display the elapsed percentage of the tasks in the results of the search.
  4. The process, case and task details sections were modified to display all the SLA information: SLA level, due date, time to limit and elapsed percentage.
  5. SLA details widget. This widget displays the information regarding all SLA in a process.
  • SLA dashboard. This new operational dashboard shows the consolidated information about the SLA for a process type. Therefore, there are several filters for selecting the process type, the status and the closing dates. For the selected filters, it displays:
    1. Two pie charts with the information of the SLA levels by processes and tasks within the selected process type;
    2. Two line charts with the information of processes and tasks closed in the last 6 months by SLA level;
    3. Performance table with average information regarding the execution of the processes for the selected process type. This table displays for each subprocess and task:
      1. The number of ocurrences in each SLA level;
      2. The average duration;
      3. The average time to limit;
      4. The average elapsed percentage.
