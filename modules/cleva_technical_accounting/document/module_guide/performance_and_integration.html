  • Module guides

    This section provides all module guides organized by module.

    • Cleva Analytics

      This module includes several data marts generated based on the business of Life, Non-life and Accounting areas. It allows extracting, from massive data volumes, information that is relevant for a quick decision-making process.

    • Cleva Frontend factory

      This module provides features to define the UX/UI of all Cleva software. It allows insurers to define menus, pages and sections, and to integrate different Cleva modules within the same user interface.

    • Cleva Outputs

      This module supports integration between Cleva software and third-party applications regarding output management, allowing document generation, sending and archiving.

    • Cleva Product factory

      This module includes product engineering features, which allow the insurance company to configure all the components the system needs to manage a product, including tariffs, data validation rules, risk acceptance, calculation of taxes and commissions, claim context, indemnity calculation rules and the behaviour expected in a specific distribution channel.

    • Cleva Technical accounting

      Cleva Technical accounting (TAC) is a solution designed to support the integration of the insurance companyâs core system with its accounting system.

      • Management console

        As the operations executed by TAC are completely automatic, the console provides a view to the data being processed and corresponding status, displaying the options and operations available.

      • Batch processing

        Batch processing allow processing a large volume of data, in sequential order and automatically, thus freeing the user and reducing the possibility of error.

      • IBML configuration

        IBML is an Insurance Business Modelling Language which, combined with a rules engine, allows TAC to configure the rules necessary for processing and mapping accounting information.

      • Performance and integration

        Performance issues are becoming increasingly important, and this is also true for TAC.The ever-growing volume of information to process in a very short period of time poses a continuous challenge. TAC continuous development and enhancement allows significantly improving the performance of its processes, namely event consumption and processing or database access.

        • View document accounting transactions

          In the area that allows viewing the technical documents of Cleva Life v7, it is now available a new widget that allows viewing the accounting transactions generated by TAC for the corresponding document.

        • Consumption of events by event date/time

          The goal of this feature is to enable TAC to consume and process the events by their date/time, thus preventing more recent events from being consumed before the older ones.

        • Performance enhancement

          The aim of this feature is to enhance TACâs performance in the consumption and processing of the documents/events generated by the technical areas.

        • Integration with Cleva Accounting

          The objective of this feature is to enable the integration between TAC and Cleva Accounting (FAC), thus allowing the accounting transactions produced by TAC to be entered into FAC automatically.

        • Integration with modelling software

          The objective of this feature is to equip TAC with the capacity to process information sourced from the modelling software and to convert this information into accounting transactions.

        • Improved performance of transaction search by tax year/month

          The search for transactions by tax year/month was improved in version 7.11.

        • Purge of processed/consumed data

          There is a new process available to perform the automatic cleaning of the temporary data from TACâs database.

        • Performance | Bulk inserts and multi-instance deployment

          With the goal of improving the TAC's performance when processing events, the storage of transactions in the database will now be carried out in bulk.

        • Work with Cost accounting

          This feature enables TAC to work with Cost accounting.

        • Work with Cost accounting

          This feature enables TAC to work with Cost accounting.

      • Event and notification system

        This system allows TAC to convert the events generated by the technical areas into accounting transactions. With this system, all events and notifications are defined as intended beforehand, with subsequent automatic processing in batches.

    • Cleva Workflow

      This module includes the workflow engine that orchestrates all business processes and corresponding insurance activities, from underwriting to claim management. It allows implementing activity automation according to the business rules defined by the insurance company, and the activities processed manually are available to users via the task manager.

Performance and integration

Performance issues are becoming increasingly important, and this is also true for TAC.The ever-growing volume of information to process in a very short period of time poses a continuous challenge. TAC continuous development and enhancement allows significantly improving the performance of its processes, namely event consumption and processing or database access.

For the Integration with other applications, TAC offers a set of web services that provide information and that can be used by external applications. As an alternative, TAC also allows generating Json files with the accounting transactions generated in a given period.
