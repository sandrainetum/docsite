Introduction

The aim of the Outputs component is to collect data to send to document management. To allow the data transfer between applications, this component has a structure that can be configured according to specific contexts, where the objects that make up those contexts are variables, and each variable represents a piece of information.

Thus, the contexts are composed of concepts. For instance, the Policy context is composed of a set of concepts that are required for the context.

In practice, these contexts correspond to the insurance business documents that will be available for printing. In addition to the default documents – which are available with the software installation – the insurance company will also be able to configure any additional ones that it deems necessary.

The aim of this document is to describe the existing contexts and concepts, as well as the variables available for each concept – i.e., the data that will be provided when a context that contains the concept in question is used.
